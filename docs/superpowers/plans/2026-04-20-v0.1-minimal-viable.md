# Lumina-Proxy v0.1.0 Minimal Viable Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement a working minimal viable LLM proxy with model-based routing, format conversion, token counting, and statistics persistence as specified in the PRD.

**Architecture:** Layered architecture with Axum HTTP server, authentication middleware, model-based router, format conversion layer, HTTP client, token counter, and structured logging. Each component has a single responsibility and communicates through well-defined interfaces.

**Tech Stack:** Rust + Tokio + Axum + Reqwest + Serde + tiktoken-rs + tracing

## Task 9: Implement core proxy logic (proxy.rs)

**Files:**
- Create: `src/proxy.rs`

- [ ] **Step 1: Define `ProxyState` struct**

Holds:
- `Config`, `reqwest::Client`, `Option<StatsWriter>`

- [ ] **Step 2: Implement `build_backend_url` function**

Based on provider type and route config:
- `ollama`: `base_url` + `/api/chat`
- `anthropic`: `base_url` + `/v1/messages`
- `gemini`: `base_url` + `/v1beta/models/{model}:streamGenerateContent?alt=sse`
- `openai`/`openai-compatible`: use configured `url` directly

- [ ] **Step 3: Implement non-streaming request handling**

Flow:
1. Parse OpenAI request from body
2. Extract model, find backend
3. Convert OpenAI request to provider format
4. Count prompt tokens (async spawn)
5. Send request to backend
6. Receive full response
7. Convert provider response back to OpenAI format
8. Count completion tokens
9. Write stats to JSONL
10. Log request completion
11. Return response to client

- [ ] **Step 4: Implement streaming request handling**

Flow:
1. Up to sending request is same as non-streaming
2. Wrap the response stream in an interceptor that:
   - Parses each provider chunk
   - Converts to OpenAI chunk format
   - Sends chunk to client immediately
   - Increments token count asynchronously
3. After stream completes:
   - Get final token count
   - Write stats to JSONL
   - Log completion

- [ ] **Step 5: Implement the main Axum handler function**

`proxy_handler` that:
- Accepts POST `/v1/chat/completions`
- Gets `ProxyState` from Axum extensions
- Reads request body
- Dispatches to streaming or non-streaming handling
