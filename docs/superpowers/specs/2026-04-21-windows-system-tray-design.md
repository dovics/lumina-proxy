# Windows 系统托盘功能设计文档

**日期：** 2026-04-21  
**状态：** 设计中  
**目标：** 为 Lumina-Proxy 添加 Windows 系统托盘支持

---

## 1. 需求概述

### 1.1 功能需求
- 在 Windows 系统右下角显示托盘图标，表示代理进程正在运行
- 右键点击托盘图标弹出菜单
- 菜单显示服务器状态信息
- 菜单支持"退出"操作，优雅关闭服务器

### 1.2 非功能需求
- 跨平台编译：代码在所有平台都能编译
- 条件编译：仅 Windows 平台启用托盘功能
- 优雅关闭：退出时等待请求完成
- 零外部资源：图标嵌入二进制

## 4. 详细设计

### 4.1 托盘菜单结构

```
┌─────────────────────────────────┐
│  🟢 服务器运行中                 │
│  监听地址: 127.0.0.1:8787       │
├─────────────────────────────────┤
│  退出                            │
└─────────────────────────────────┘
```

### 4.2 优雅关闭流程

1. **用户操作**：右键托盘 → 点击"退出"
2. **事件捕获**：Tao 事件循环捕获菜单点击事件
3. **信号发送**：通过 `tokio::sync::mpsc::Sender` 发送关闭信号
4. **信号接收**：主线程 Receiver 收到信号
5. **优雅关闭**：调用 `axum::serve` 的 `with_graceful_shutdown()`
6. **等待完成**：等待所有正在处理的请求完成
7. **进程退出**：清理资源后正常退出

### 4.3 条件编译策略

```rust
// src/tray.rs
#[cfg(windows)]
pub mod tray {
    // Windows 托盘实现
}

#[cfg(not(windows))]
pub mod tray {
    // 空实现或 stub
    pub async fn run_tray(/* ... */) -> Result<()> {
        Ok(())
    }
}
```

### 4.4 Cargo.toml 依赖配置

```toml

```

## 5. 实现步骤

### 阶段 1：基础框架
1. 添加条件编译依赖到 Cargo.toml
2. 创建 src/tray.rs 模块骨架
3. 在 lib.rs 中导出 tray 模块

### 阶段 2：托盘实现
1. 实现菜单构建（muda）
2. 实现托盘图标创建（tray-icon）
3. 实现 Tao 事件循环
4. 实现通道通信（mpsc）

### 阶段 3：集成
1. 修改 main.rs 集成托盘线程
2. 实现优雅关闭信号处理
3. 配置 Axum 服务器的 graceful shutdown

### 阶段 4：测试
1. Windows 平台编译测试
2. Linux/macOS 平台编译验证（无托盘但能编译）
3. 功能测试：启动、退出、优雅关闭

## 6. 错误处理

### 6.2 事件循环异常
