# Windows 系统托盘功能设计文档

**日期：** 2026-04-21  
**状态：** 设计中  
**目标：** 为 Lumina-Proxy 添加 Windows 系统托盘支持

---

## 1. 需求概述

### 1.1 功能需求
- 在 Windows 系统右下角显示托盘图标，表示代理进程正在运行
- 右键点击托盘图标弹出菜单
- 左键点击：无操作（第一版暂不实现）
- 菜单显示服务器状态信息
- 菜单支持"退出"操作，优雅关闭服务器

### 1.2 非功能需求
- 跨平台编译：代码在所有平台都能编译
- 条件编译：仅 Windows 平台启用托盘功能
- 优雅关闭：退出时等待请求完成
- 零外部资源：图标嵌入二进制

## 3. 架构设计

### 3.1 整体架构

**关键说明：** 在 Windows 上，Tao 事件循环必须运行在**主线程**（用于 Windows 消息循环），Axum 服务器运行在后台线程。

```
┌─────────────────────────────────────────────────────────────┐
│                       主线程 (Windows)                       │
│  ┌──────────────┐      ┌──────────────────┐                │
│  │  Tao 事件循环│◀─────│  mpsc Sender     │◀───┐           │
│  │              │      │                  │    │           │
│  └──────┬───────┘      └──────────────────┘    │           │
│         │                                      │           │
│  ┌──────▼───────┐      ┌──────────────────┐    │           │
│  │  Tray Icon   │◀────▶│   Muda 菜单      │    │           │
│  │  (Windows)   │      │                  │    │           │
│  └──────────────┘      └──────────────────┘    │           │
└───────────────────────────────────┬────────────┼───────────┘
                                    │            │
                                    │ 通道通信    │
                                    │            │
┌───────────────────────────────────▼────────────┼───────────┐
│                   后台 Tokio 线程               │           │
│  ┌──────────────┐      ┌──────────────────┐    │           │
│  │  Axum 服务器 │◀─────│  Tokio 运行时    │    │           │
│  │              │      │                  │    │           │
│  └──────────────┘      └─────────┬────────┘    │           │
│                                   │             │           │
│                          ┌────────▼────────┐    │           │
│                          │  mpsc Receiver  │────┘           │
│                          └──────────────────┘                │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 双向通道设计

- **Shutdown 通道**：`mpsc::channel<()>` 托盘 → 服务器（关闭信号）
- 注：第一版服务器状态是静态的（启动后不变），暂不需要状态更新通道

### 3.2 模块结构

```

├── main.rs          # 修改：集成托盘
├── lib.rs           # 修改：导出 tray 模块
└── ...
```

## 4. 详细设计

### 4.1 托盘菜单结构

```
┌─────────────────────────────────┐
│  🟢 服务器运行中                 │
│  监听地址: 127.0.0.1:8787       │
├─────────────────────────────────┤
│  退出                            │
└─────────────────────────────────┘
```

### 4.2 优雅关闭流程

1. **用户操作**：右键托盘 → 点击"退出"
2. **事件捕获**：Tao 事件循环（主线程）捕获菜单点击事件
3. **信号发送**：通过 `tokio::sync::mpsc::Sender` 发送关闭信号（非阻塞）
4. **信号接收**：后台 Tokio 线程的 Receiver 收到信号
5. **优雅关闭**：调用 `axum::serve` 的 `with_graceful_shutdown()`
6. **等待完成**：等待所有正在处理的请求完成
7. **资源清理**：TrayIcon 和 Menu 对象被 drop，`tray-icon` 库自动清理系统托盘图标
8. **进程退出**：清理资源后正常退出

**幂等性保证**：多次点击"退出"不会造成问题，通道关闭后发送失败会被静默忽略

### 4.4 条件编译策略

```rust
// src/tray.rs
#[cfg(windows)]
pub mod tray {
    // Windows 托盘实现
}

#[cfg(not(windows))]
pub mod tray {
    // 空实现或 stub
    pub async fn run_tray(/* ... */) -> Result<()> {
        Ok(())
    }
}
```

### 4.3 图标资源

**图标文件位置：** `assets/icon.ico`
- 格式：ICO（Windows 原生格式）
- 尺寸：包含 16x16, 32x32, 48x48（支持高 DPI）
- 使用 `include_bytes!` 宏嵌入二进制

**图标嵌入方式：**
```rust
// Windows 平台使用 include_bytes! 嵌入图标
#[cfg(windows)]
const TRAY_ICON: &[u8] = include_bytes!("../assets/icon.ico");
```

**高 DPI 支持：** `tray-icon` 库内部处理高 DPI 显示，无需额外代码。

### 4.5 Cargo.toml 依赖配置

```toml
# 仅 Windows 平台添加托盘相关依赖
[target.'cfg(windows)'.dependencies]
tray-icon = "0.15"
muda = "0.12"
tao = "0.28"
```

注：图标嵌入使用 Rust 内置的 `include_bytes!` 宏，无需额外 `include_dir` 依赖。

**未来扩展性：** 如需添加动态状态更新（如连接数、请求数等），可添加反向 mpsc 通道（服务器 → 托盘）实现菜单内容实时更新。

## 5. 实现步骤

### 阶段 1：基础框架

1. 修改 `main.rs` 线程模型：

2. 实现优雅关闭信号处理
3. 配置 Axum 服务器的 graceful shutdown

## 6. 错误处理

### 6.1 托盘创建失败
- 记录警告日志
- 服务器继续正常运行（降级处理）
- 用户仍可通过正常方式关闭服务器

### 6.2 事件循环异常
- 记录错误日志
- 不影响服务器运行（服务器在后台线程继续运行）

### 6.3 通道发送失败
- 关闭信号发送失败时记录错误
- 触发强制关闭作为后备方案

### 6.4 图标资源清理
- `tray-icon` 库在对象 drop 时自动从系统托盘移除图标
- 即使进程异常退出，Windows 也会在鼠标悬停时清理残留图标
